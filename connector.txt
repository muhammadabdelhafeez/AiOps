# Integration Redevelopment Guide ŌĆö BMC, SCOM, AppDynamics, vROps, EMCO, Lansweeper, and AI Analysis

**Last updated:** 2026-06-17
**Purpose:** This guide explains how to rebuild the current multi-source infrastructure analysis system on another server/system. It consolidates connector configuration, API URLs, request formats, authentication requirements, data fields, and the AI model pipeline.

> Security note: this guide intentionally uses environment-variable names and placeholders. Do **not** hard-code API keys, passwords, access secrets, webhook URLs, or client secrets in source code or Markdown.

```text

```

**Headers:**

```http
Content-Type: application/json

```

**Body:**

```json

  "access_key": "${BMC_ANALYSIS_BMC_ACCESS_KEY}",
  "access_secret_key": "${BMC_ANALYSIS_BMC_ACCESS_SECRET_KEY}"
}
```

**Expected response fields:**

```json

  "json_web_token": "<jwt-token>",
  "user_id": "<user>",
  "tenant_name": "<tenant>"
}
```

### 4.3 Event Search Request

**Method:** `POST`
**URL:**

```text
{bmc.base-url}/events-service/api/v1.0/events/msearch
```

**Headers:**

```http
Content-Type: application/json
Accept: application/json
Authorization: Bearer <json_web_token>
```

**Body:**

```json
{
  "size": 100,
  "query": {
    "bool": {
      "filter": [
        {
          "range": {
            "creation_time": {
              "gte": "now-60m",
              "lte": "now"
            }
          }
        },
        {
          "query_string": {
            "analyze_wildcard": true,
            "query": "!severity:OK"
          }
        }
      ]
    }
  },
  "sort": {
    "creation_time": {
      "order": "desc",
      "unmapped_type": "boolean"
    }
  },
  "script_fields": {}
}
```

For pagination, add `search_after` from the last hit sort token:

```json
{
  "search_after": ["<last-sort-token>"]
}
```

Typical fields used by the analysis:

```text

_source.alert_name
_source.msg
_source.class
_source.status
_source.priority
```

### 4.5 Development Notes

- Deduplicate by event `_id` when available.
- If `_id` is missing, use index + sort + source hash.
- Fetch both open and closed alerts from the time window.
- Keep full BMC `msg`; do not truncate before AI analysis.

## 6. Microsoft SCOM Integration

SCOM is not consumed through a REST API in the current project. It uses PowerShell Remoting / WinRM to execute SCOM cmdlets on the management server.

### 6.1 Configuration

```properties
scom.enabled=true
scom.management-server=<scom-management-server-fqdn>
scom.username=${BMC_ANALYSIS_SCOM_USERNAME:}
scom.password=${BMC_ANALYSIS_SCOM_PASSWORD:}
scom.domain=<domain>
scom.hours-back=1
scom.connection-timeout-seconds=60
scom.verify-ssl=true
scom.winrm-port=5986
scom.use-https=true
scom.auth-method=Kerberos
```

Current default server in this project:

```text
dcvscoap12.corp.kfh.kw
```

WinRM URL shape:

```text
https://{scom.management-server}:5986/wsman
```

If using non-HTTPS WinRM:

```text
http://{scom.management-server}:5985/wsman
```

### 6.2 Local PowerShell Invocation Pattern

The Java process starts `powershell.exe` and runs `Invoke-Command`.

```powershell
$ErrorActionPreference = 'Stop'
$secPass = ConvertTo-SecureString '<password>' -AsPlainText -Force
$cred = New-Object System.Management.Automation.PSCredential('<domain>\<username>', $secPass)
$startLocal = (Get-Date).AddHours(-4) # configured hours + timezone buffer

$result = Invoke-Command `
  -ComputerName '<scom-management-server>' `
  -Port 5986 `
  -UseSSL `
  -Credential $cred `
  -Authentication Kerberos `
  -ScriptBlock {
    param([datetime]$startUtcInner)

    Import-Module OperationsManager -ErrorAction Stop | Out-Null
    New-SCOMManagementGroupConnection -ComputerName $env:COMPUTERNAME | Out-Null

    $alerts = @(
      Get-SCOMAlert |
      Where-Object { $_.TimeRaised -ge $startUtcInner } |
      Select-Object `
        Id, Name, Severity, Priority, Category, ResolutionState, `
        @{Name='MonitoringObjectName'; Expression = {$_.MonitoringObjectDisplayName}}, `
        MonitoringObjectPath, NetbiosComputerName, TimeRaised, TimeResolved, LastModified, Description
    )

    $alerts | Sort-Object TimeRaised -Descending
  } `
  -ArgumentList $startLocal

if ($null -eq $result -or $result.Count -eq 0) {
  Write-Output '[]'
} else {
  $result | ConvertTo-Json -Depth 10 -Compress
}
```

### 6.3 Important SCOM Fields to Normalize

```text
Id
Name
Severity
Priority
Category
ResolutionState
MonitoringObjectName
MonitoringObjectPath
NetbiosComputerName
TimeRaised
TimeResolved
LastModified
Description
```

### 6.4 Development Notes

- SCOM may return alert times in local time; the current code over-fetches by +3 hours and filters precisely in Java.
- Deduplicate by `Id` because SCOM alerts can update multiple times within the same window.
- Strip non-printable control characters from PowerShell JSON before parsing.
- Required server setup:
  - WinRM enabled on management server.
  - User has permission to run SCOM PowerShell cmdlets.
  - `OperationsManager` PowerShell module is installed on the SCOM server.
  - Kerberos/WinRM SPNs and firewall rules are correct.

### 8.3 KFH Query

```sql
SELECT
    h.[label],
    h.[host],
    h.[description] AS host_description,
    e.[title],
    e.[severity],
    e.[description] AS event_description,
    DATEADD(HOUR, 3, e.[event_time]) AS event_time_kuwait
FROM [db_owner].[tb_host_events] AS e
JOIN [db_owner].[tb_hosts] AS h
    ON e.[host_id] = h.[id]
WHERE
    e.[event_time] >= DATEADD(
        MINUTE,
        -?,
        DATEADD(HOUR, -3, SYSDATETIME())
    )
    AND e.[title] IN ('Connection Quality', 'Host State')
    AND (
        h.[description] LIKE 'Server Support%'
        OR h.[description] LIKE 'Call Center%'
        OR h.[description] LIKE 'Database%'
        OR h.[description] LIKE 'DevOps - OpenShift%'
        OR h.[description] LIKE 'Storage & Sun%'
        OR h.[description] LIKE 'Telephony%'
    )
ORDER BY e.[event_time] DESC;
```

Parameter `?` is `minutesBack`.

### 8.4 CCTV Query

```sql
SELECT
    h.[label],
    h.[host],
    h.[description] AS host_description,
    e.[title],
    e.[severity],
    e.[description] AS event_description,
    DATEADD(HOUR, 3, e.[event_time]) AS event_time_kuwait
FROM [dbo].[tb_host_events] AS e
JOIN [dbo].[tb_hosts] AS h
    ON e.[host_id] = h.[id]
WHERE
    e.[event_time] >= DATEADD(
        MINUTE,
        -?,
        DATEADD(HOUR, -3, SYSDATETIME())
    )
    AND e.[title] IN ('Connection Quality', 'Host State')
    AND h.[label] NOT LIKE 'CCTV%'
ORDER BY e.[event_time] DESC;
```

### 8.5 Important EMCO Fields to Normalize

```text
label
host
host_description
title
severity
event_description
event_time_kuwait
source_domain: KFH / CCTV
```

### 8.6 Development Notes

- Execute KFH and CCTV queries in parallel.
- Set `PreparedStatement#setQueryTimeout` using `emco.query-timeout-seconds`.
- Convert EMCO UTC event time to Kuwait time using `DATEADD(HOUR, 3, e.event_time)`.

### 17.3 BMC Helix Connector Development Contract

#### Goal

Develop a REST connector that authenticates to BMC Helix, fetches recent non-OK events, paginates with `search_after`, deduplicates events, and normalizes them for AI root-cause analysis.

#### Required configuration

```properties
bmc.enabled=true
bmc.base-url=https://<bmc-tenant>.onbmc.com
bmc.access-key=${BMC_ANALYSIS_BMC_ACCESS_KEY:}
bmc.access-secret-key=${BMC_ANALYSIS_BMC_ACCESS_SECRET_KEY:}
bmc.login-endpoint=/ims/api/v1/access_keys/login
bmc.events-endpoint=/events-service/api/v1.0/events/msearch
bmc.minutes-back=60
bmc.max-events=500
bmc.page-size=100
bmc.timeout-seconds=120
```

#### Implementation steps

1. Validate `base-url`, `access-key`, and `access-secret-key`.
2. `POST` login request to `{base-url}{login-endpoint}`.
3. Extract `json_web_token` from the response.
4. Build event query for `creation_time >= now-{minutesBack}m` and `!severity:OK`.
5. `POST` query to `{base-url}{events-endpoint}` with `Authorization: Bearer <jwt>`.
6. Collect hits from response.
7. If the response includes hit `sort`, continue with `search_after` until:
   - no hits are returned,
   - max events is reached,
   - sort token stops advancing,
   - or the returned page is smaller than page size.
8. Deduplicate by native `_id`; fallback to index + sort + source hash.
9. Normalize to `NormalizedEvent`.

#### BMC fields the AI needs

```text
source_hostname, source_address, _service_name, alert_name, msg, class, status, priority, mc_object, severity, creation_time, modification_time
```

#### BMC normalization rules

| Normalized field | BMC source |
|---|---|
| `source` | `BMC` |
| `sourceEventId` | hit `_id` or source alert/event id if present |
| `timestamp` | `creation_time` |
| `severity` | `_source.severity` or `_source.priority` |
| `status` | `_source.status` |
| `host` | `_source.source_hostname` |
| `ipAddress` | `_source.source_address` |
| `service` | `_source._service_name` |
| `title` | `_source.alert_name` |
| `message` | full `_source.msg` |
| `component` | `_source.mc_object` or `_source.class` |

#### Edge cases

- Login returns HTTP 401/403.
- Login response has no `json_web_token`.
- Events endpoint returns malformed JSON.
- Events endpoint returns duplicate hits.
- Events endpoint returns no `sort` token.
- Event message is very long; keep raw message for AI but summarize only for UI if needed.

#### Acceptance criteria

- Connector returns an empty list when disabled.
- Connector fetches at least one valid event when BMC has data.
- Connector masks access key/secret and JWT in logs.
- Connector does not crash other connectors on failure.
- AI input includes full BMC message text and exact host/service names.

### 17.4 vROps / Aria Operations Connector Development Contract

#### Goal

Develop a REST connector that acquires a vROps token, fetches alert pages, filters by time/status, enriches alerts with resource details, and normalizes them for AI analysis.

#### Required configuration

```properties
vrops.enabled=true
vrops.host=<vrops-host-or-ip>
vrops.username=${BMC_ANALYSIS_VROPS_USERNAME:}
vrops.password=${BMC_ANALYSIS_VROPS_PASSWORD:}
vrops.auth-source=<auth-source-name>
vrops.page-size=1000
vrops.max-pages=200
vrops.hours=1
vrops.workers=12
vrops.verify-ssl=true
vrops.timeout-seconds=120
```

#### Implementation steps

1. Build base URL: `https://{vrops.host}/suite-api/api`.
2. Acquire token with `POST /auth/token/acquire`.
3. Store token in memory; optionally cache token in a local secure file.
4. Fetch alerts using `GET /alerts?page={page}&pageSize={pageSize}&_no_links=true`.
5. Add header `Authorization: vRealizeOpsToken <token>`.
6. Filter alerts by `startTimeUTC`, optionally `updateTimeUTC` and `cancelTimeUTC`.
7. Stop pagination on empty page, old page, max pages, or invalid pagination.
8. Collect unique `resourceId` values.
9. Enrich using `GET /resources/{resourceId}?_no_links=true`.
10. Normalize to `NormalizedEvent`.

#### vROps fields the AI needs

```text
id, alertDefinitionName, alertDefinitionId, criticality, status, startTimeUTC, updateTimeUTC, cancelTimeUTC, resourceId, resourceKey.name, resourceKey.adapterKindKey, resourceKey.resourceKindKey, alertType, subType, alertImpact, resourceHealth, resourceHealthValue
```

#### vROps normalization rules

| Normalized field | vROps source |
|---|---|
| `source` | `vROps` |
| `sourceEventId` | alert `id` |
| `timestamp` | `startTimeUTC` converted from epoch millis |
| `severity` | `criticality` |
| `status` | `status` |
| `host` | `resourceKey.name` when resource is VM/host |
| `component` | `resourceKindKey` / `adapterKindKey` |
| `title` | `alertDefinitionName` |
| `message` | alert definition + impact + resource details |

#### Edge cases

- Token API returns HTML instead of JSON.
- Token response has no `token` field.
- Alert page returns invalid pagination error.
- Alert references a resource that no longer exists.
- Resource enrichment fails; keep original alert anyway.
- SSL verification may fail in development; keep SSL bypass behind config only.

#### Acceptance criteria

- Connector renews or reacquires token when expired.
- Connector can fetch multiple pages.
- Connector enriches resource names and health where possible.
- Connector continues with partial data when enrichment fails.
- AI input includes resource name, resource kind, criticality, and alert definition.

### 17.5 SCOM Connector Development Contract

#### Goal

Develop a WinRM/PowerShell connector that executes SCOM cmdlets on the management server, returns JSON, sanitizes it, deduplicates alerts, and normalizes them for AI analysis.

#### Required configuration

```properties
scom.enabled=true
scom.management-server=<scom-management-server-fqdn>
scom.username=${BMC_ANALYSIS_SCOM_USERNAME:}
scom.password=${BMC_ANALYSIS_SCOM_PASSWORD:}
scom.domain=<domain>
scom.hours-back=1
scom.connection-timeout-seconds=60
scom.winrm-port=5986
scom.use-https=true
scom.auth-method=Kerberos
scom.verify-ssl=true
```

#### Implementation steps

1. Build full username as `{domain}\{username}` unless username already contains domain or UPN.
2. Build a local PowerShell command using `powershell.exe -NoProfile -NonInteractive -ExecutionPolicy Bypass`.
3. Create a PSCredential from username/password.
4. Use `Invoke-Command` to run the remote SCOM script.
5. Remote script imports `OperationsManager` module.
6. Remote script runs `New-SCOMManagementGroupConnection`.
7. Remote script runs `Get-SCOMAlert` and selects required fields.
8. Convert result to JSON with `ConvertTo-Json -Depth 10 -Compress`.
9. Extract JSON from PowerShell output.
10. Remove invalid control characters.
11. Parse JSON and deduplicate by `Id`.
12. Normalize to `NormalizedEvent`.

#### SCOM fields the AI needs

```text
Id, Name, Severity, Priority, Category, ResolutionState, MonitoringObjectName, MonitoringObjectPath, NetbiosComputerName, TimeRaised, TimeResolved, LastModified, Description
```

#### SCOM normalization rules

| Normalized field | SCOM source |
|---|---|
| `source` | `SCOM` |
| `sourceEventId` | `Id` |
| `timestamp` | `TimeRaised` or current state-change time if available |
| `severity` | `Severity` |
| `status` | `ResolutionState` |

### 17.6 AppDynamics Connector Development Contract

```properties
appdynamics.enabled=true

```

#### Implementation steps

1. Create WebClient/HTTP client with controller base URL.

4. For each app, optionally in parallel:
   - Fetch request snapshots for errors.
   - Fetch health-rule violations.
   - Fetch request snapshots for slow transactions.
5. For request snapshots, keep records where:
   - `userExperience == ERROR`, or
   - `errorOccurred == true`, or
   - `userExperience in [SLOW, VERY_SLOW, STALL]` for slow transactions.
6. For violations, support both response formats:
   - array,
   - object with `policy-violations`.
7. Extract snapshot `id` as Snapshot ID.
8. Extract violation `id` as Violation ID.
9. Build `sourceEventId` from `Violation ID`, `Snapshot ID`, or `Alert ID`.
10. Normalize to `NormalizedEvent`.

#### AppDynamics fields the AI needs

```text
Application ID, Application Name, Alert ID, Snapshot ID, Violation ID, Violation Name, Alert Type, Severity, Status, Start Time, End Time, Description, Affected Entity Name, Affected Entity Type, Business Transaction ID, Business Transaction Name, Tier ID, Tier Name, Node Name, URL, Deep Link URL, Time Taken, Error Details, Error Summary, Stack Trace
```

#### AppDynamics normalization rules

| Normalized field | AppDynamics source |
|---|---|
| `source` | `AppDynamics` |
| `sourceEventId` | `Violation ID`, `Snapshot ID`, or `Alert ID` |
| `timestamp` | `serverStartTime` or `startTimeInMillis` |
| `severity` | `severity`, `userExperience`, or computed status |
| `status` | `status` or `userExperience` |
| `host` | `nodeName` or `affectedEntityName` if server-like |
| `application` | `applicationName` |
| `service` | `tierName` or `affectedEntityName` |
| `component` | tier/node/business transaction |
| `title` | violation name, error summary, or alert type |
| `message` | full error details, description, stack trace, and transaction details |
| `urlOrEndpoint` | URL or business transaction name |

#### Mandatory AppDynamics Event ID rule for AI

Every AppDynamics record passed to AI must expose a visible ID field:

```text
AppDynamics Event ID: <Alert ID / Violation ID / Snapshot ID or Not specified in alert data>
```

In final reports, correlation maps, clear root-cause chains, and correlated alert chains must render AppDynamics links like:

```text
[AppDynamics 20:55] <application/component> ŌĆö <short alert text> (Event ID: <id>)
```


#### Edge cases

- Connector preserves Application ID and Event ID.

## 18. Key Security Rules

- Never log passwords, access secrets, API keys, JWT tokens, or Basic Auth headers.
- Mask secrets in debug logs.
- Use least-privilege service accounts.
- Use HTTPS and certificate verification where possible.
- For temporary SSL bypass in development, isolate it behind config and never default it to insecure mode in production.
- Store credentials in environment variables, Vault, Key Vault, or a secrets manager.
- Rotate keys after moving to a new system.
