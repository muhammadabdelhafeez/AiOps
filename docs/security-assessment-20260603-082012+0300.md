## 3. Endpoint assessment and fixes

### `ConnectorController` — `src/main/java/org/aiopsanalysis/controller/ConnectorController.java`

Applicable endpoints:
- `GET /api/connectors`
- `GET /api/connectors/{connectorId}`
- `POST /api/connectors`
- `PUT /api/connectors/{connectorId}`
- `DELETE /api/connectors/{connectorId}`
- `POST /api/connectors/test`
- `GET /api/connectors/types`

Findings and actions:
- **CWE-20:** Pageable/sort were user-controlled. Fixed in `ConnectorService` by clamping page size to 100 and allowlisting sort fields: `name`, `type`, `enabled`, `createdAt`, `updatedAt`.
- **CWE-20 / SSRF:** Connector create/test configs accept URLs and paths. Fixed by DTO bounds and plugin validation for BMC paths, numeric config values, auth type, TLS verification, and SSRF validator behavior.
- **CWE-200:** Raw `IllegalArgumentException` messages could be reflected. Fixed with generic `Request validation failed` response.
- **Multi-tenancy:** Existing repository calls are tenant-scoped. API context filter now rejects missing/invalid `X-Tenant-Id` and `X-User-Id` on `/api/**` before controller execution.
- **RBAC:** Service methods are annotated with permission-based `@PreAuthorize`; production still requires an authenticated authority source from SSO/gateway to make those annotations effective.

### `IncidentController` — `src/main/java/org/aiopsanalysis/controller/IncidentController.java`

Applicable endpoints:
- `GET /api/v1/incidents`
- `GET /api/v1/incidents/app/{appId}`
- `GET /api/v1/incidents/{incidentId}`
- `POST /api/v1/incidents/{incidentId}/acknowledge`
- `POST /api/v1/incidents/{incidentId}/close`
- `GET /api/v1/incidents/{incidentId}/topology`
- `GET /api/v1/incidents/updated?hoursAgo={n}`
- `GET /api/v1/incidents/reopened?hoursAgo={n}`
- `GET /api/v1/incidents/stats`

Findings and actions:
- **OWASP A01 / IDOR / CWE-862/CWE-863:** `GET /{incidentId}`, `acknowledge`, and `close` previously used incident ID only. Fixed by requiring `X-Tenant-Id` and calling tenant-scoped repository lookups through `IncidentService`.
- **CWE-20:** `hoursAgo` accepted negative or excessive values. Fixed with `@Min(1)` and `@Max(168)`.
- **CWE-20:** Notes were unbounded and allowed control characters. Fixed with max length 2000 and control-character rejection except CR/LF/TAB.
- **Multi-tenancy:** All listed endpoints now require both tenant and user headers, with UUID format validation by the API context filter and controller-level validation.

## 9. Residual risks / follow-up backlog

1. **Production authentication/RBAC integration:** `SecurityConfig` still permits static/UI/API routes and connector RBAC annotations require an authenticated authority source to be effective. Integrate SSO/JWT or a trusted gateway authentication filter that maps verified identities and permissions to Spring Security authorities.
2. **CSRF posture:** CSRF is disabled. Keep disabled only for stateless header/token APIs. Re-enable or add CSRF tokens if cookie/browser session auth is introduced.
3. **Audit completeness:** Connector writes create audit logs. Incident acknowledge/close create status history entries; if the enterprise audit table is mandatory for every write, add `AuditLogRepository` integration to incident writes in a follow-up.
4. **Crypto master key:** `CryptoUtil` can generate a random development key when `INSIGHT_MASTER_KEY` is absent. Production should fail closed or enforce secret injection via deployment profile.
5. **Frontend/static JS:** A complete frontend XSS/accessibility/API-client pass was not performed beyond backend/API-consuming controls in this assessment.
6. **Full test suite noise:** Full tests completed, but existing startup logs include H2 metadata and generated dev security password warnings. Consider cleaning test profile/autoconfiguration to reduce noise in CI evidence.
